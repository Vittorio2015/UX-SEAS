Overview

This Accessibility evaluation was not conducted with any user of assistive technology. It is highly recommended to test the product with target users, as well as with users of assistive technology.

Select any of the images for a full scale view.

Accessibility Evaluation

Specifications

This accessibility audit was conducted on a MacBook Pro with a Macintosh operating system version 10.14.5.

The product was accessed with a Chrome browser via https://eaf-qa.seas.harvard.edu/app/dashboard/confirmations

The testing tools used for this audit were:

  • VoiceOver to simulate screen reader technology
  • Chrome Vox browser extension to simulate screen reader technology
  • Google Inspect for code inspection
  • Siteimprove Chrome browser extension for code inspection
  • WebAIM Color Contrast Checker to check the validity of the color contrast for visual impairment

Summary

An inspection of the content revealed several accessibility issues.
Even though most of the required A and AA WCAG 2.1 guidelines were detected, only testing with native users of assistive technology will ensure that all the content is properly accessible.
User testing should be scheduled after all the accessibility issues listed below are addressed.

Dashboard / Confirmations tab

Finding
The color of the text and the color of the background are not in sufficient contrast to each other.
a.menu-item--active, a.menu-item--active:hover, a.menu-item--active:visited,
a.menu-item--active:link {
color: #181818;
}
Recommendation
The contrast ratio should be at least 4.5:1 for normal text and 3:1 for large text.
Use a lighter color with sufficient contrast.
More information on color contrast.
a.menu-item--active, a.menu-item--active:hover, a.menu-item--active:visited,
a.menu-item--active:link {
color: #C4C4C4;
}
Finding
The WAI-ARIA image, created with role="img", does not have an accessible name.
<svg aria-hidden="true"
data-prefix="fas" data-icon="check"
class="svg-inline--fa fa-check fa-w-16 "
role="img"
xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512"
style="">
<path fill="currentColor" d="..." style=""></path>
</svg>
Recommendation
Images can be effective way to convey meaning, such as to provide additional information to text content or to assign labels to buttons. Text alternatives are vital for people who can’t see them.
  • Provide appropriate text alternatives for images
  • Make sure icon fonts have accessible text alternatives
  • If the image is not relevant and it doesn't need any description, provide an empty alt tag (alt="")

More information on accessible images

EAF Details tab

Finding
Although visible, Approval Status elements don't have any accessible information.

<div class="status-tracker__status-event status--approved" style="">Approval</div>
<div class="status-tracker__status-event status--denied" style="">Approval</div></div>

Recommendation
Add the same information shown on mouseover to each element.

<div class="status-tracker__status-event status--approved" style="">
Approval
<span hidden>EAF approved by ... on 04-05-2019</span>
</div>
<div class="status-tracker__status-event status--denied" style="">
Approval
<span hidden>EAF denied by ... on 04-05-2019</span>
</div>


Learn more about Invisible Content Just for Screen Reader Users
Finding
A grid is used instead of a table
Recommendation
If you can use a native HTML element [HTML51] or attribute with the semantics and behavior you require already built in, instead of re-purposing an element and adding an ARIA role, state or property to make it accessible, then do so.
Avoid to disjoint tables, even if the content is dynamically added or altered (sorting)
Learn more about the First Rule of ARIA Use
